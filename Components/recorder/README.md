# OEDISI Recorder

Data recording federate for capturing HELICS co-simulation outputs in efficient columnar formats.

## Overview

The recorder federate subscribes to HELICS publications and saves time-series data to disk in multiple formats. It provides:
- Real-time co-simulation data recording
- Efficient columnar storage using Apache Arrow/Feather format
- CSV output for compatibility
- Configurable subscriptions
- Automatic data aggregation

## Architecture

- **Recorder**: Main federate class (record_subscription.py)
- **FastAPI Server**: REST API wrapper (server.py)
- **Output Formats**: Feather (PyArrow) and CSV

## Key Features

- ✅ HELICS subscription recording
- ✅ Feather format output (PyArrow columnar storage)
- ✅ CSV format output for compatibility
- ✅ Time-series data aggregation
- ✅ Configurable output paths
- ✅ Efficient memory management
- ✅ Low-overhead recording

## Usage

### Running the Server

```python
# Using the entry point
recorder-server

# Or directly with Python
python server.py
```

The server runs on port 5679 (configurable via `PORT` environment variable).

### Python API

```python
from recorder import Recorder

# Create and run recorder federate
recorder = Recorder(broker_config)
recorder.run()
```

### Configuration

The recorder is configured via:
- **Component configuration**: Subscription list and output settings
- **HELICS broker config**: Connection details and timing
- **Input mapping**: Links between publications and recorded variables

Configuration files are typically generated by the OEDISI broker.

### Output Files

The recorder creates two output files:
- `{output_name}.feather` - Apache Feather format (efficient, fast)
- `{output_name}.csv` - CSV format (human-readable, compatible)

Both files contain the same data with columns:
- `time` - Simulation time
- `{subscription_1}` - First subscribed variable
- `{subscription_2}` - Second subscribed variable
- ... (one column per subscription)
